## Таблица атрибутов и настроек JSON

| Атрибут/настройка | Что делает | Пример |
|-------------------|------------|--------|
| `[JsonPropertyName("name")]` | Задаёт имя поля в JSON | `"name"` вместо `"Name"` |
| `[JsonIgnore]` | Поле не попадает в JSON | Пароли, внутренние поля |
| `PropertyNamingPolicy.CamelCase` | Все поля в camelCase | `"realName"` вместо `"RealName"` |
| `JsonStringEnumConverter` | Enum как строка | `"Marvel"` вместо `0` |
| `WriteIndented = true` | JSON с отступами | Для удобства при разработке |

## Главные выводы

### 1. Что такое сериализация и десериализация

**Сериализация** — это процесс преобразования объекта C# в JSON-строку для передачи по сети.
**Десериализация** — это обратный процесс: преобразование JSON-строки обратно в объект C#.

### 2. Зачем нужен [JsonIgnore] — реальные примеры

```csharp
    public class User
    {
        public string Login { get; set; }
        
        [JsonIgnore]
        public string PasswordHash { get; set; }  //  Не должен попасть в JSON!
    }
```
### 3. Почему enum лучше сериализовывать строкой, а не числом

1. Понятность: Клиент сразу видит смысл значения («Marvel» вместо загадочного «0»), не нужно лезть в документацию.
2. Надёжность: Если вы измените порядок значений в enum-е, числа поменяются и сломают клиентское приложение, а строковые имена останутся прежними.
3. Читаемость: В логах, Swagger и при отладке гораздо удобнее воспринимать имена, чем цифры.
